# EEG Motor Imagery Prediction Application

A Streamlit application for classifying imagined left-hand vs right-hand fist
movements from EEG signals, using the
[PhysioNet EEG Motor Movement/Imagery Dataset](https://www.physionet.org/content/eegmmidb/1.0.0/).

Built for the **ARISE Summer Research Internship** technical challenge —
*Towards Explainable Multimodal AI for Fatigue Detection Using EEG and Wearable Sensors*.

## Pipeline

### 1. Dataset

| Setting | Value |
|---------|-------|
| Dataset | PhysioNet EEG Motor Movement/Imagery (v1.0.0) |
| Subjects | S001–S040 |
| Runs | 4, 8, 12 (imagined left/right fist) |
| Annotations | T1 = left fist, T2 = right fist, T0 = rest (ignored) |

### 2. Preprocessing

| Step | Detail |
|------|--------|
| Channel standardisation | `mne.datasets.eegbci.standardize` + standard_1005 montage |
| Band-pass filter | 8–30 Hz, zero-phase FIR (`firwin`) |
| Event extraction | T1 / T2 from EDF annotations via `mne.events_from_annotations` |
| Epoch window | **+1.0 s → +4.0 s** relative to cue onset (skips initial evoked response; captures sustained motor imagery ERD) |
| Baseline correction | None |

### 3. Feature Extraction

Two complementary feature sets are extracted from each epoch:

#### Band-power features (for SVM / RF / MLP)
Channels: 9 sensorimotor channels — `FC3, FCz, FC4, C3, Cz, C4, CP3, CPz, CP4`

For each channel, Welch PSD → mean power in each band → log transform:

| Band | Range |
|------|-------|
| Mu / Alpha | 8–13 Hz |
| Beta | 13–30 Hz |

Additionally, a **C3/C4 laterality index** `(C4 − C3) / (C4 + C3)` is appended
for each band. This is the primary biomarker for left vs. right hand motor imagery
(contralateral ERD causes C4 power to drop for left-hand imagery and vice versa).

Total: `9 channels × 2 bands + 2 laterality = 20 features` per epoch.

#### CSP features (for CSP-SVM / CSP-LDA)
All 64 EEG channels are used. **Common Spatial Patterns (CSP)** learns 6 spatial
filters that maximise the variance ratio between classes, producing 6 log-variance
features per epoch. CSP is the field-standard method for motor imagery
classification and consistently outperforms fixed-channel band-power approaches
by 15–25 percentage points in cross-subject settings.

### 4. Models

All classifiers are wrapped in a scikit-learn `Pipeline`:

| Model | Input | Classifier |
|-------|-------|-----------|
| `svm` | Band-power (20 feat.) | StandardScaler → SVC (RBF, C=10) |
| `random_forest` | Band-power (20 feat.) | StandardScaler → RandomForest (500 trees) |
| `mlp` | Band-power (20 feat.) | StandardScaler → MLP (256-128-64, early stopping) |
| `csp_svm` | Raw epochs (64 ch.) | CSP (6 components) → StandardScaler → SVC (RBF, C=10) |
| `csp_lda` | Raw epochs (64 ch.) | CSP (6 components) → LDA (shrinkage="auto") |

### 5. Evaluation

| Setting | Value |
|---------|-------|
| Split | Stratified 80 / 20 hold-out |
| Cross-validation | 5-fold CV on training split |
| Metrics | Accuracy, weighted F1, confusion matrix, classification report |

Results are saved to `models/model_metadata.json` and `results/offline_model_results.csv`,
and are displayed in the app's *Offline Model Training Summary* section.

## Run Locally

### Prerequisites

- Python 3.10–3.13
- ~1 GB free disk space (for 40 subjects of PhysioNet data)

### 1. Clone and install

```bash
git clone <your-repo-url>
cd eeg_prediction_application

python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. Train models offline

```bash
python train_models.py
```

This downloads subjects S001–S040 (cached in `./data/`), trains all 5 models,
and saves them to `models/`. Expected runtime: 30–90 minutes depending on your CPU.

Sample output:

```
======================================================================
  PhysioNet EEG MMI — offline model training
======================================================================

Building dataset from subjects 1–40 on runs [4, 8, 12] ...
  → Subject S001: downloading runs [4, 8, 12]...
      kept 45 epochs (23 L / 22 R)
  ...

  Training svm (band-power features) ...
    accuracy = 0.570  |  f1 = 0.568  |  cv = 0.558 ± 0.023

  Training csp_svm (CSP features, all channels) ...
    accuracy = 0.720  |  f1 = 0.718  |  cv = 0.705 ± 0.031

  Training csp_lda (CSP features, all channels) ...
    accuracy = 0.710  |  f1 = 0.708  |  cv = 0.698 ± 0.028
```

### 3. Launch the app

```bash
streamlit run app.py
```

Open the URL Streamlit prints (typically <http://localhost:8501>).

### 4. Try it

1. Download a sample EDF, e.g. subject S001 run 4:
   `https://physionet.org/files/eegmmidb/1.0.0/S001/S001R04.edf`
2. Upload it using the *Upload EDF file* widget in the sidebar.
3. Select a model (recommend `csp_svm` or `csp_lda` for best accuracy).
4. Inspect the visualisations, offline summary, and per-epoch predictions.

## Deploy on Streamlit Community Cloud

1. Commit the trained `models/*.pkl` and `models/model_metadata.json` to the repo.
2. Push to GitHub.
3. Go to [share.streamlit.io](https://share.streamlit.io), sign in, create a new app
   pointing at `app.py`.
4. Set Python version to 3.11 in the Streamlit Cloud settings.
5. Wait for the build, then paste the public URL into this README and the submission form.

> The EEG dataset is **not** needed at deployment — the app only reads the pre-trained
> `.pkl` files. Uploaded EDF files are processed on the fly with the same pipeline
> used at training time.
